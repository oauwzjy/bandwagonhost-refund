## 搬瓦工退款政策是什么

搬瓦工（BandwagonHost）提供 **30 天退款保证**，覆盖账户内所有付款的全额退还。不是按套餐退，是按账户退——意思是你账户下有几台 VPS，退款时一起清掉，一分不留地全部退回来。

这个退款是"全账户退款"，不支持只退某一台机器。

## 搬瓦工退款条件：6 条全部要满足

这是直接从搬瓦工官方服务条款里扒出来的，6 个条件缺一不可：

**1. 必须是新订单，且在 30 天内**

注意这里说的 30 天，是**账户注册时间**，不是购买 VPS 的时间。如果你的账户很早就注册了，最近才买 VPS，那对不起，30 天的窗口早过了，退款页面会直接告诉你不符合条件。

**2. 账户信誉良好**

没有欠款、没有违反过服务条款，这条一般人都没问题。

**3. 没有发起过争议或退款**

历史上没有通过 PayPal 申请过争议，也没有在当前或关联账户下用过退款权利。每个账户**只有一次退款机会**，用完就没了，退款之后可以继续购买服务，但无法再次申请退款。

**4. IP 没有被列入黑名单**

分配给你 VPS 的 IP 地址，不在任何黑名单里，而且之前没有因为被封而申请过免费换 IP。如果 IP 已经被封，是不能退款的。

**5. 月流量使用不超过配额的 10%**

这个坑很多人没意识到。VPS 用着用着流量刷了一大半，再想退——系统直接拒绝。实际体验下来，10% 是个很紧的限制，基本上买来试了一两天不行就得决定，时间拖长了流量容易超。

**6. 退款申请不能有滥用嫌疑**

没有在多个关联账户重复申请退款，整体退款行为正常。

## 退款之前必须做的一件事

**备份数据。**

提交退款申请之后，账户下所有 VPS 的数据、快照、备份会被**立即、不可逆地删除**。这不是一个可以撤销的操作，也不是"我回复 I agree 之后再慢慢备份"——提交申请那一刻系统就开始清理了。

如果 VPS 上有重要的东西：网站文件、数据库、配置文件、密钥，先全部拉下来，确认完整，再去申请退款。

## 退款到账时间和退款方式

退款会退回到**原支付方式**：

- **PayPal 付款**：退回 PayPal 账户，一般当天到账
- **支付宝付款**：退到搬瓦工账户余额（Credit），不是退到支付宝，这个余额可以用来以后买新服务，但不能提现

支付宝这条很多人没注意——退款不是直接回到支付宝，是留在平台余额里。如果只是想换个套餐，这其实挺方便的；如果真的想把钱取出来，那就麻烦了，基本取不出来。

## 退款常见失败原因

总结一下最容易翻车的几种情况：

- **账户注册超过 30 天**：最常见。很多人注册账号很早，买 VPS 是后来的事，等到想退款时窗口已经过了
- **流量用超了 10%**：试用了一段时间，VPS 跑了不少流量，退款条件就不满足了
- **IP 被封过**：VPS 的 IP 地址被封、或者申请过免费换 IP，这种情况不能退款
- **之前用过退款**：账户历史上申请过退款，再申请会被拒
- **通过 PayPal 发起了争议**：一旦走 PayPal 争议流程，账户会被立即暂停，后续处理更复杂，搬瓦工还会按每小时 $40 收取处理费

说实话，搬瓦工的退款政策对新用户相对友好，但时间窗口很紧，流量限制也挡住了一部分人。真正想试水的话，建议买来之后快速测一下线路表现和稳定性，两三天内做决定比较稳。

## 常见问题

**Q：退款是退某一台 VPS，还是全账户一起退？**

全账户退款，账户下的所有 VPS 全部清掉，不能只退其中一台。如果你有多台机器只想退一台，通过退款流程是做不到的，只能联系客服走正常取消流程（但那不是退钱，只是停服）。

**Q：30 天是从购买 VPS 开始算，还是从注册账户开始算？**

从账户注册时间算。这是最容易搞混的一条——不是从买 VPS 那天算的。

**Q：支付宝付款退款后，钱会回到支付宝吗？**

不会直接退回支付宝。退款会进入搬瓦工账户的 Credit 余额，可以用于下次购买，但不能提现。

**Q：退款失败怎么办？**

登录后台查看工单里的具体原因。如果是流量超了但你觉得计算有误，可以在工单里回复并提供截图让客服核实。如果确实不符合条件，基本就是正常的申请拒绝，走 PayPal 争议不是个好选项，反而会让情况更麻烦。

**Q：能不能用 PayPal 争议来代替退款？**

不建议。走 PayPal 争议会导致账户下所有服务被立即暂停，搬瓦工还会收取每小时 $40 的处理费，服务要等欠款结清才能恢复。得不偿失。
